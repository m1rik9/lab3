## Задание 1

**Условие:**
Напишите функцию, определяющую, является ли последовательность палиндромом. Реализовать две версии: с использованием рекурсии и без. Функция должна работать с любыми последовательностями (списки, строки, кортежи).
**Описание проделанной работы:**
В ходе выполнения лабораторной работы я реализовал две функции для проверки последовательности на палиндром.

Для итеративной функции я использовал два указателя: left на начало и right на конец последовательности. В цикле while left < right сравниваю элементы на этих позициях. При несовпадении возвращаю False, иначе сдвигаю указатели навстречу друг другу. Если цикл завершился, возвращаю True.

Для рекурсивной функции определил базовый случай: если длина последовательности ≤ 1, возвращаю True. В рекурсивном случае сравниваю первый и последний элементы. Если они не равны — возвращаю False, если равны — рекурсивно вызываю функцию для подпоследовательности без первого и последнего элементов.

В итоге я получил две рабочие функции, решающие задачу разными способами. Итеративная версия эффективнее по памяти, рекурсивная — нагляднее.

Реализованы две независимые программы, каждая со своей функцией `is_palindrome`. Программы не используют глобальные переменные, все данные передаются через аргументы и локальные переменные.
```python
def is_palindrome_iterative(seq):
    """Итеративная проверка палиндрома."""
    left, right = 0, len(seq) - 1
    while left < right:
        if seq[left] != seq[right]:
            return False
        left += 1
        right -= 1
    return True


def is_palindrome_recursive(seq):
    """Рекурсивная проверка палиндрома."""
    if len(seq) <= 1:
        return True
    if seq[0] != seq[-1]:
        return False
    return is_palindrome_recursive(seq[1:-1])


# Проверка работы функций
print(is_palindrome_iterative([1, 2, 3, 2, 1]))
print(is_palindrome_iterative('spam'))
print(is_palindrome_recursive([1, 2, 3, 2, 1]))
print(is_palindrome_recursive('spam'))

```

## Задание 2
**Условие:**
Напишите функцию для вычисления последовательности, заданной рекуррентным соотношением:
xi=xi−1+xi−3x_i = x_{i-1} + x_{i-3}xi​=xi−1​+xi−3​
с начальными условиями: x1=x2=x3=1x_1 = x_2 = x_3 = 1x1​=x2​=x3​=1. Реализовать две версии: с использованием рекурсии и без.
**Описание проделанной работы:**
В ходе выполнения лабораторной работы я реализовал две функции для вычисления n-го члена последовательности.

Для итеративной функции: если n ≤ 3, возвращаю 1. Иначе создаю переменные x1, x2, x3 = 1, 1, 1. В цикле от 4 до n вычисляю x_next = x3 + x1 и сдвигаю значения: x1, x2, x3 = x2, x3, x_next. По завершении возвращаю x3.

Для рекурсивной функции: базовый случай — если n ≤ 3, возвращаю 1. В рекурсивном случае возвращаю sequence_recursive(n - 1) + sequence_recursive(n - 3).

В итоге я получил две рабочие функции. Итеративная версия работает за O(n) и эффективна для любых n. Рекурсивная версия имеет экспоненциальную сложность и подходит только для малых n (до 30-35).
```python
def sequence_iterative(n):
    """Итеративное вычисление n-го члена."""
    if n <= 0:
        return None
    if n <= 3:
        return 1
    
    x1, x2, x3 = 1, 1, 1
    for i in range(4, n + 1):
        x_next = x3 + x1
        x1, x2, x3 = x2, x3, x_next
    return x3


def sequence_recursive(n):
    """Рекурсивное вычисление n-го члена."""
    if n <= 0:
        return None
    if n <= 3:
        return 1
    return sequence_recursive(n - 1) + sequence_recursive(n - 3)


# Проверка работы функций
print("n = 1:", sequence_iterative(1), sequence_recursive(1))
print("n = 2:", sequence_iterative(2), sequence_recursive(2))
print("n = 3:", sequence_iterative(3), sequence_recursive(3))
print("n = 4:", sequence_iterative(4), sequence_recursive(4))
```
